# B2C 로또 예측 서비스 프론트엔드 UI/UX 기획서

본 기획서는 사용자(B2C)가 접속하여 가장 직관적으로 결과를 확인하고 예측 시스템을 통제할 수 있는 **프론트엔드 화면 구성과 라우팅(탐색) 흐름**에 집중하여 작성되었습니다.

---

## 1. 글로벌 UI/UX 컨셉 (Design System)

### 1.1 로딩 상태 정책
*   **초기 페이지 로드**: 모든 페이지에 스켈레톤 UI 적용. 실제 컨텐츠와 동일한 레이아웃 형태의 플레이스홀더를 표시.
*   **무한 스크롤 추가 로드**: 리스트 하단에 스피너 표시.
*   **API 에러**: 에러 메시지 텍스트 표시. 별도 에러 페이지로 이동하지 않음.

*   **테마**: 다크 모드(Dark Mode). 형광/네온 포인트 컬러(블루, 민트, 옐로우).
*   **스타일**: 글래스모피즘(Glassmorphism) 기반 투명 카드.
*   **동작**: 마이크로 애니메이션, 세로 스크롤 및 좌우 스와이프 지원 (Mobile First).

---

## 2. 화면 구조 및 GNB (글로벌 네비게이션바) 흐름

사용자의 심리적 흐름(결과 확인 ➡️ 예측 참여 ➡️ 분석)에 맞춰 GNB 순서를 **[메인(Rounds)] ➡️ [예측(Predict)] ➡️ [통계(Stats)] ➡️ [서재(Saved)]** 순으로 배치합니다.

*   **PC**: 상단 고정 헤더 GNB.
*   **모바일**: 하단 고정 탭바. 각 탭은 아이콘 + 텍스트 레이블로 표시.

### 2.1 메인 랜딩 페이지 (`/` ➡️ `/rounds` 매핑)
사이트 접속 시 가장 먼저 보이는 첫 화면(Home)을 '오피셜 당첨 결과' 페이지로 설정합니다.

*   **목적**: 사용자가 가장 많이 검색하는 정보인 '이번 주 당첨 번호'를 최우선으로 보여주어 유입 즉시 만족감을 제공하고 신뢰성 확보.
*   **데이터 소스**: PLN-predict (FastAPI) `/lotto` 엔드포인트에서 호출.

#### 화면 구성

**회차 리스트**
*   최신 회차부터 순서대로 카드 형태로 나열. 별도 히어로 영역 없음.
*   카드 표시 정보: 회차 번호, 추첨일, 당첨 번호 볼 6개 + 보너스 볼 1개. (당첨금·당첨자 수는 노출하지 않음)
*   첫 번째 카드(최신 회차)는 테두리나 배경 강조로 시각적으로 구분.
*   **무한 스크롤** 방식으로 과거 데이터를 순차 로드. 별도 인터랙션(필터·검색·상세 페이지 이동) 없음.

#### 번호 볼 디자인

다크 글래스모피즘 테마에 맞춰 채도를 낮추고 네온 포인트 컬러 계열로 통일. 기존의 원색 조합 대신 각 구간을 **테마 포인트 컬러(블루, 민트, 옐로우)의 명도 변형**으로 표현하여 전체 디자인 시스템과 일관성 유지.

| 번호 구간 | 컬러 방향 | 비고 |
|-----------|-----------|------|
| 1 – 10    | 딥 블루 계열 (예: `#3B82F6` 계통의 다크 톤) | 포인트 블루 기반 |
| 11 – 20   | 민트/청록 계열 (예: `#06B6D4` 계통) | 포인트 민트 기반 |
| 21 – 30   | 옐로우/앰버 계열 (예: `#EAB308` 계통의 머티드 톤) | 포인트 옐로우 기반 |
| 31 – 40   | 슬레이트/뉴트럴 계열 (예: `#64748B`) | 중립 톤으로 대비 |
| 41 – 45   | 로즈/코럴 계열 (예: `#F43F5E` 계통의 다크 톤) | 소수 번호 강조 |
| 보너스     | 테두리만 골드(`#FBBF24`)로 강조, 볼 배경은 반투명 처리 | 별도 시각 구분 |

*   볼 내부는 단색 채우기 대신 **미세한 그라디언트 또는 내부 글로우(glow)** 효과를 적용해 글래스모피즘 테마와 조화.
*   볼 배경색은 다크 배경 위에서 충분한 명도 대비를 확보하면서도 형광·원색이 되지 않도록 채도를 조절.

### 2.2 예측 및 시스템 검증 허브 (`/predict.tsx`)
번호를 생성하는 액션 창과 과거 시스템 성과를 검증하는 창을 동일한 페이지 내의 **3개의 탭(Tab)**으로 배치하여 자연스러운 전환을 유도합니다.

*   **페이지 내부 탭 (Inner Tabs) 구성 및 순서**:
    1.  `[이번 주 번호 생성]` (기본 진입 탭)
    2.  `[추천 제외수 이력]` (시스템 검증 1)
    3.  `[추천 가중치 이력]` (시스템 검증 2)
*   **URL 쿼리로 탭 상태 유지** (`?tab=generate` / `?tab=exclude` / `?tab=weight`): 새로고침·뒤로가기 시 마지막 탭 복원. stats 페이지와 동일한 방식.

*   **A. [이번 주 번호 생성] 탭 (메인 기능 3단 패널)**
    *   **패널 1 (제외 설정)**: 1~45번을 7열 격자형 그리드(로또 용지 형태)로 표시. `추천 제외수 불러오기` 및 수동 토글 가능.
        *   **제외수 시각 처리**: 볼의 구간별 색상은 유지하되 `opacity: 0.25` 수준으로 밝기를 낮추고 볼 위에 X 아이콘 오버레이. 색상 충돌 없이 어떤 구간 번호든 동일한 방식으로 "제외됨" 상태 표현.
    *   **패널 2 (가중치 조작)**: 1~45번을 7열 격자형 그리드로 표시. 각 볼 오른쪽에 가중치 수치와 +/- 버튼 배치.
        *   **가중치 범위**: 0 ~ 100 정수. 합산이 100일 필요 없으며 각 번호가 독립적인 선호도 값을 가짐.
        *   **조작 단위**: +1 / -1. 0 미만, 100 초과는 불가.
        *   `추천 가중치 불러오기` 버튼으로 PLN-predict 추천값 자동 세팅 가능.
        *   드로우 시 제외수를 제외한 나머지 번호들의 가중치를 기반으로 상대적 확률 계산.
    *   **패널 3 (드로우 액션)**:
        *   상단: 생성 수량 입력창 + `[생성하기]` 버튼 + `[엑셀 다운로드]` 버튼.
        *   하단: 생성된 번호 조합을 카드 단위로 `flex-wrap` 배치. 카드가 패널 너비를 초과하면 자동으로 다음 줄로 이동. 결과가 많을 경우 패널 내부 스크롤.
        *   **결과 필터**: 결과 카드 목록 바로 위에 위치. 생성 후 결과를 실시간으로 좁혀보는 필터. 재생성 없이 즉시 적용. 각 필터는 선택적으로 활성화.
            *   **홀짝 비율**: 홀수 개수 범위 설정 (예: 홀 2~4개).
            *   **고저 비율**: 고수(23~45) 개수 범위 설정 (예: 고 2~4개).
            *   **번호합 범위**: 6개 번호 합계의 최솟값·최댓값 설정.
            *   **AC값**: 허용할 AC값 범위 설정 (0~9).
            *   **고정수**: 반드시 포함된 번호 기준 필터. 7열 격자형 그리드에서 토글.
        *   각 카드: 번호 볼 6개 + `[서재에 담기]` 버튼.
*   **B. [추천 제외수 이력] 탭**
    *   최신 회차부터 역순으로 카드 나열. 무한 스크롤.
    *   **카드 표시 정보**: 회차 번호, 해당 회차의 추천 제외수 볼 목록을 1열로 나열.
    *   **좁은 화면(모바일) 대응**: C탭과 동일하게 카드 내 번호 리스트 영역에 가로 스크롤 적용.
    *   **볼 시각 처리**:
        *   제외수 볼: 31-40 구간 슬레이트(`#64748B`)보다 더 어두운 회색 배경 + 볼 텍스트(숫자) 색상도 별도 지정(예: `#94A3B8` 수준의 낮은 명도). 비활성 느낌 강조.
        *   제외수 중 실제 당첨번호에 포함된 볼: 구간별 원래 색상으로 복원하여 "제외했는데 당첨된" 번호를 직관적으로 식별.
    *   **데이터 소스**: PLN-predict 신규 엔드포인트 (개발 예정).
*   **C. [추천 가중치 이력] 탭**
    *   최신 회차부터 역순으로 카드 나열. 무한 스크롤.
    *   **카드 표시 정보**: 회차 번호, 1~45번 전체를 가중치 내림차순으로 정렬하여 1열로 나열. 각 볼 오른쪽에 가중치 수치 표시.
    *   **볼 시각 처리**: B탭과 동일. 기본은 어두운 회색 + 낮은 명도 텍스트. 실제 당첨번호에 포함된 볼만 구간별 원래 색상으로 복원.
    *   **좁은 화면(모바일) 대응**: 카드 내 번호 리스트 영역에 가로 스크롤 적용. 카드 자체는 전체 너비를 유지하고 내부 리스트만 스크롤되도록 처리.
    *   **데이터 소스**: PLN-predict 신규 엔드포인트 (개발 예정).

### 2.3 심층 통계 분석 대시보드 (`/stats.tsx`)
단순 예측을 넘어서, 통계 지표를 전문적으로 확인하는 데이터 공간입니다.

#### 전역 필터
모든 탭에 공통으로 적용되는 필터. **스크롤 시 상단 고정(sticky)** — GNB 바로 아래에 붙어 항상 노출.
*   **회차 범위**: 시작 회차 ~ 종료 회차 직접 입력 + 슬라이더로 범위 조작 가능. 입력값과 슬라이더 양방향 동기화.
*   **단축 버튼**: 최근 50 / 최근 100 / 최근 200 버튼으로 빠른 범위 설정.
*   **보너스 번호**: 포함/제외 토글.

#### 탭 구성 (기본 진입: 출현빈도)
URL 쿼리(`?tab=...`)로 탭 상태 유지 — 새로고침·뒤로가기 시 복원.

1.  `[출현빈도]`: 1~45번 각 번호의 누적 출현 횟수. 내림차순 정렬 + 바 차트.
2.  `[핫&콜드]`: 핫넘버(최근 자주 나온) + 콜드넘버(오래 안 나온) 분리 표시.
3.  `[번호합]`: 당첨번호 6개 합계의 구간별 분포. 바 차트 + 비율(%).
4.  `[홀짝/고저]`: 홀짝 비율 분포 + 고저 비율 분포 + 색상(구간)별 출현 분포.
5.  `[끝수]`: 당첨번호 일의 자리(0~9) 출현 빈도. 바 차트 + 비율(%).
6.  `[번호쌍]`: 동시 출현 TOP 20 번호 조합. 바 차트.
7.  `[AC값]`: 번호 조합의 다양성 지표(0~9) 분포. 바 차트 + 비율(%).
8.  `[마킹패턴]`: 번호별 출현 빈도를 로또 용지 격자(9열) 히트맵으로 시각화. 진할수록 자주 나온 번호.

### 2.4 내 서재 (`/saved.tsx`)
*   **핵심 기능**: `/predict`에서 저장한 번호 조합 보관소.
*   **저장 방식**: 로컬스토리지. 최대 1000게임. 로그인 기능 추가 시 마이페이지로 분리하고 API 호출 방식으로 전환 예정.
*   **카드 레이아웃**: `flex-wrap` 배치. 카드 너비를 초과하면 자동으로 다음 줄로 이동.
*   **카드 표시 정보**: 번호 볼 6개 + 개별 삭제 버튼.
*   **전체 삭제**: 목록 상단에 전체 삭제 버튼 배치.
*   **당첨 확인 기능 없음**: 번호만 저장하므로 회차 연동 불필요.
